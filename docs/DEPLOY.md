# Reborn50 上线部署指南

## 1. 前置条件

| 步骤 | 负责人 | 预计耗时 |
|------|--------|---------|
| 注册小游戏账号 | 运营 | 1~3 天 |
| 获取 AppID | 运营 | 即时 |
| 开发者绑定 | 运营 | 即时 |
| 实名认证 | 运营 | 1~5 天 |
| **小游戏备案** | 运营 | **10~25 天** |
| 配置 downloadFile 域名 | 开发 | 即时 |

> 备案与开发**并行进行**。项目启动第一天就提交备案，避免后期阻塞。

## 2. 备案流程

1. 登录 [mp.weixin.qq.com](https://mp.weixin.qq.com)
2. 设置 → 小程序备案 → 填写备案信息
3. 平台初审（1-2 个工作日）
4. 工信部短信核验（24 小时内完成）
5. 通管局审核（1-20 个工作日）
6. 备案成功 → 获得备案号

## 3. 代码上传

```
微信开发者工具 → 点击「上传」
  ├── 版本号：v1.0.0（遵循 semver）
  ├── 备注：本次更新内容
  └── 确认上传
```

上传成功后在 MP 后台「版本管理」可看到开发版本。

## 4. 体验版验证

1. MP 后台 → 版本管理 → 选开发版本 → 设为「体验版」
2. 添加测试微信号（最多可添加 100 个）
3. 测试人员扫码体验
4. 按 [TESTING.md](./TESTING.md) 执行全量测试

## 5. 提交审核

### 5.1 准备材料

| 材料 | 要求 | 状态 |
|------|------|------|
| 游戏内容介绍 | 说明玩法、规则、特点 | [ ] |
| 游戏截图 | >= 4 张，像素风格 | [ ] |
| 游戏自审自查报告 | MP 下载模板，填写盖章 | [ ] |
| 版本更新说明 | 描述本次更新内容 | [ ] |
| 著作权证明（可选） | 软著或自研声明 | [ ] |

### 5.2 提审步骤

1. MP 后台 → 版本管理 → 提交审核
2. 选择游戏引擎（输入「其他引擎」→「微信小游戏原生」）
3. 填写游戏内容介绍
4. 上传截图
5. 上传自审自查报告
6. 填写版本更新说明
7. 确认提交

### 5.3 审核时长

- **IAA（广告变现）游戏**：13~37 个工作日
- **IAP（内购）游戏**：9~22 个工作日

本游戏当前为无变现免费游戏，属于 IAA 类别。

### 5.4 加急审核

满足以下条件可申请（每季度 2 次）：
- 已有历史通过版本
- 累计注册用户 > 5000
- 用于紧急 BUG 修复或重大活动

## 6. 发布上线

1. 审核通过后，微信服务通知会提醒
2. MP 后台 → 审核版本 → 点击「提交发布」
3. 选择「全量发布」
4. 确认发布

## 7. 发布后监控

- 实时关注 MP 后台数据：DAU、留存、崩溃率
- 用户反馈渠道：客服消息、评价
- 每 1-2 周迭代版本修复问题

## 8. 版本迭代

遵循语义化版本号 `MAJOR.MINOR.PATCH`：

| 版本 | 内容 |
|------|------|
| v1.0.0 | 首版上线 |
| v1.1.0 | 功能更新（新怪物、新装备） |
| v1.1.1 | Bug 修复 |
| v2.0.0 | 重大更新（排行榜、好友系统） |
